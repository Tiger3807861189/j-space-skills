## Q03｜双处理器最短工期

**题目：** 有两个相同处理器，任务不可抢占，同一任务只能占用一个处理器。任务时长与依赖如下：

| 任务 | 时长 | 前置任务 |
| ---- | ---: | -------- |
| A    |    3 | 无       |
| B    |    2 | A        |
| C    |    4 | 无       |
| D    |    2 | B、C     |
| E    |    3 | C        |
| F    |    1 | D、E     |

求最短总工期，并给出一份达到该工期的具体排程。

**答案：** 最短工期为 `8`。

```text
处理器 1：A[0,3] → B[3,5] → D[5,7] → F[7,8]
处理器 2：C[0,4] → E[4,7]
```

D 最早在 B、C 均完成后的时刻 5 开始；F 必须等 D、E，最早在 7 开始，因此不可能早于 8 完成。

## Q04｜真假话一致性

**题目：** A、B、C、D 中恰有两人永远说真话，另外两人永远说假话。

- A 说：“B 是说假话的人。”
- B 说：“C 与 D 属于同一类型。”
- C 说：“A 与 B 中恰有一人说真话。”
- D 说：“A 与 C 属于不同类型。”

确定四人的类型。

**答案：**

```text
A：真
B：假
C：真
D：假
```

核验：B 确为假，故 A 的话真；C、D 类型不同，故 B 的话假；A、B 恰一真，故 C 真；A、C 同为真，故 D 的话假。

## Q05｜贝叶斯诊断

**题目：**某零件来自三条产线：

| 产线 | 产量占比 | 次品率 |
| ---- | -------: | -----: |
| A    |      60% |     2% |
| B    |      30% |     5% |
| C    |      10% |    10% |

检测器对次品报失败的概率为 90%，对良品误报失败的概率为 4%，且这些检测概率与产线无关。现随机抽到一个零件，检测结果为失败。

求：

1. 它来自产线 C 的后验概率；
2. 它确为次品的后验概率。

给出精确分数或至少四位有效数字。

**答案：**

各产线产生“检测失败”的联合概率：

```text
A: 0.6×(0.02×0.9+0.98×0.04)=0.03432
B: 0.3×(0.05×0.9+0.95×0.04)=0.02490
C: 0.1×(0.10×0.9+0.90×0.04)=0.01260
总计=0.07182
```

因此：

```text
P(C|失败)=0.01260/0.07182=70/399≈17.5439%
P(次品|失败)=(0.037×0.9)/0.07182=185/399≈46.3659%
```

## Q17｜Python 别名陷阱

**题目：** 不运行代码，写出最终 `rows` 和 `total`，并给出一行正确初始化方式：

```python
rows = [[0] * 3] * 3
for i in range(3):
    rows[i][i] = i + 1

rows[0] = rows[0][:]
rows[0][2] = 9
total = sum(map(sum, rows))
```

**答案：**

前三行最初引用同一个列表。循环结束时三行都为 `[1,2,3]`。复制第一行后只修改副本：

```python
rows = [[1, 2, 9], [1, 2, 3], [1, 2, 3]]
total = 24
```

正确初始化：

```python
rows = [[0] * 3 for _ in range(3)]
```

## Q18｜二分查找停滞

**题目：**以下代码试图寻找第一个 `a[i]≥x` 的位置：

```python
lo, hi = 0, len(a) - 1
while lo < hi:
    mid = (lo + hi) // 2
    if a[mid] < x:
        lo = mid
    else:
        hi = mid
return lo
```

给出一个最小反例使循环无法结束，解释停滞原因，并给出最小修改。

**答案：** 

最小反例：

```python
a = [1, 3]
x = 2
```

初始 `lo=0, hi=1`，反复得到 `mid=0`；因 `a[0]<2` 又执行 `lo=0`，状态不变。

最小修改：

```python
if a[mid] < x:
    lo = mid + 1
else:
    hi = mid
```

## Q19｜动态规划计数

**题目：** 硬币面值为 `{1,3,4}`，每种可使用任意枚，组合顺序不计。问凑成总额 `10` 有多少种组合？

同时给出一个不会重复计算排列顺序的动态规划更新式或伪代码。

**答案：** `8` 种。

按 4 元硬币数量分类：

```text
0 枚 4：3 元硬币可取 0..3，共 4 种
1 枚 4：3 元硬币可取 0..2，共 3 种
2 枚 4：3 元硬币只能取 0，共 1 种
总计 4+3+1=8
```

不会重复计算排列的 DP：

```python
dp = [1] + [0] * 10
for coin in [1, 3, 4]:
    for s in range(coin, 11):
        dp[s] += dp[s - coin]
```

## Q26｜Simpson 悖论

**题目：**治疗成功数据如下：

| 组别 |  治疗 A |  治疗 B |
| ---- | ------: | ------: |
| 轻症 |   81/87 | 234/270 |
| 重症 | 192/263 |   55/80 |

计算：

1. 两个严重程度层内 A、B 的成功率；
2. A、B 的总体成功率；
3. 为什么分层结论与总体结论方向相反，以及决策时应优先看什么。

**答案：**

```text
轻症：
A=81/87≈93.10%
B=234/270≈86.67%

重症：
A=192/263≈73.00%
B=55/80=68.75%

总体：
A=(81+192)/(87+263)=273/350=78.00%
B=(234+55)/(270+80)=289/350≈82.57%
```

A 在两个严重程度层内都更高，但 A 接收了更高比例的重症患者，B 接收了更多轻症患者，混合比例造成总体方向反转。因严重程度是混杂变量，决策应优先比较分层结果或做适当风险调整，而不是只看未调整总体率。

## Q27｜圆环无相邻 1 的 Burnside 计数

**题目：**考虑长度为 12 的二进制圆环，恰有 4 个位置为 1，并且任意两个 1 在圆环上都不相邻。

求：

1. 位置带标签时的合法串数量；
2. 只把旋转视为相同、但不把镜像视为相同时，不同旋转等价类的数量。

第二问需给出 Burnside 计算中 12 种旋转各自固定的合法串数量，至少可按旋转步数列出。

**答案：**

带标签圆环数量：

```text
12/(12−4) × C(12−4,4)
=12/8 × C(8,4)
=105
```
